Тест по Test Case
Что такое Test Case?
  • Это код, который разработчики пишут для проверки функций.
  • Это документ, который описывает шаги, ожидаемые результаты и условия для проверки конкретной функциональности или поведения ПО.
  • Это отчет, который содержит информацию об ошибках, найденных в приложении.
  • Это документ с описанием дизайна программного обеспечения.
Что должно быть в хорошем тест-кейсе?
  • Четкие шаги для выполнения, ожидаемый результат и четкие условия прохождения/неудачи теста.
  • Только название тест-кейса и дата создания.
  • Описание всех возможных сценариев использования системы.
  • Спецификация системы, включая архитектуру и все модули.
Сколько тест-кейсов может быть на проекте?
  • Тест-кейсов всегда должно быть ровно столько, сколько функций в проекте.
  • Количество тест-кейсов не имеет ограничений и зависит от сложности и требований к проекту.
  • Всегда достаточно 10–15 тест-кейсов для любого проекта.
  • Должен быть только один тест-кейс для каждого модуля в проекте.
